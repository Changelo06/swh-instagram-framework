PATCH 5 — Add Database + User History (Cloud)

Requires Patch 4 done first. This is your biggest lift.


 Create a free Supabase project at supabase.com
 Set up tables:

users — managed by Supabase Auth
scrape_runs — stores each run per user (profile URLs, date, results)
results — stores scraped data linked to each run


 Enable Row-Level Security (RLS) so users only see their own data
 Enable automated daily backups in Supabase dashboard
 Connect your backend to Supabase using the Supabase JS or Python SDK
 Build user auth flow:

 Signup page
 Login page
 Session persistence (stay logged in)


 On each scrape run — save results to DB linked to logged-in user
 Build a Run History view in the app dashboard
